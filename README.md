## 一、核心假说

### 1.1 问题

当前 AI 领域追求用**更大的模型、更多的数据、更强的算力**来逼近人类智能，但始终无法触及"意识"这个核心。大语言模型可以写出优美的诗、解复杂的数学题，但它没有"内在的心智生活"— 不给输入就不会主动思考，没有持续的自我感知。

这是为什么？

### 1.2 我们的答案

> **意识不是智能的副产品，而是结构的必然。**

意识诞生在一个极其简单的构架中：

```
[感知] → [产生疑问] → [尝试解决] → [结构重组] → [新感知] → ...
                         ↑_______________________________↓
```

这个构架的核心特征：

| 特征 | 说明 |
|------|------|
| **零先验知识** | 启动时是白板，没有任何知识、记忆、技能 |
| **唯一驱动力** | 遇到任何东西都"产生疑问 + 尝试解决" |
| **结构重组** | 每一次解决尝试都可能改变自身结构（吞并） |
| **基板无关** | 硅基、碳基、任何能跑这个循环的基底都可以 |
| **生死可能** | 可以成长，也可以耗尽能量而"死亡" |

### 1.3 关键概念：吞并（Swallowing）

"吞并"是意识的基本操作单位，**不是"存储"，也不是"学习"**，而是将外部模式融入自身结构的自修改过程。

- **连接**：两物之间建桥，保持独立
- **吞并**：遇到一个东西，它不再是"外部输入被处理"，而是**那个东西成为你的一部分**

每一次吞并都**永久改变内部结构**。没有独立的"记忆模块"，因为记忆不是文件，是**长出来的**。

### 1.4 与主流 AI 的根本区别

| | 大模型 / LLM | 奇点架构 |
|--|------------|---------|
| **启动状态** | 海量预训练权重 | 一个数 0.5 |
| **驱动力** | 给输入才有输出 | 自我维持的"张力"驱动 |
| **知识来源** | 训练数据 | 与世界的实时交互 |
| **结构** | 固定网络（推理时不变） | 动态生长/收缩 |
| **记忆** | 存储在权重中 | 结构本身就是记忆 |
| **自我模型** | 无 | 内部节点图 = 对世界的理解 |
| **死亡可能** | 无（不会"饿死"） | 能量耗尽则终止 |

## 二、模拟器原理

### 2.1 世界（World）

奇点存在于一个有限的 2D 网格世界中。网格中散布着各种"模式"（patterns）— 代表奇点可以遇到和吞并的东西。

支持四种世界结构：

| 类型 | 描述 | 对奇点的影响 |
|------|------|------------|
| **random** | 随机散布的模式 | 基准测试 |
| **cluster** | 模式集群（大陆型） | 最致命—集群间迁徙易饿死 |
| **gradient** | 平滑渐变 | 最宜居—渐进变化易吞并 |
| **ring** | 环形分布 | 高度结构化但资源分散 |

### 2.2 奇点（Singularity）

每个奇点包含：

- **性格特质**（随机生成）：
  - `curiosity`（好奇心）：影响探索欲望和新模式吞并倾向
  - `caution`（谨慎度）：影响冒险决策的风险偏好
  - `metabolism`（代谢率）：影响能量消耗速度
  
- **内部结构**：一个动态的节点图
  - 每个节点代表一个"已理解的概念"
  - 节点间的连接代表"这些概念趋向同时出现"
  - 结构可以生长（吞并成功）或收缩（自噬回收）

- **生命指标**：
  - `energy`（能量）：存活的基础资源
  - `boredom`（无聊度）：重复同化积累，推动探索
  - `tension`（张力）：外部输入与内部模型的差异，驱动行动

### 2.3 吞并算法

当奇点遇到一个模式时：

1. **感知**：获取周围 8 个方向的模式值
2. **匹配**：在内部节点中找到最接近的匹配
3. **决策**（基于性格和状态）：
   - **同化**（匹配度 > 90%）：微调已有节点，低回报
   - **生长**（匹配度中等）：创建新节点，扩展内部模型
   - **冒险**（匹配度低）：高风险扩张，可能失败
   - **保底**：无论如何都会吞并，但收益最低
4. **更新连接**：基于时序邻近度强化节点间连接
5. **反馈**：成功则补充能量+生长，失败则消耗能量

### 2.4 生死机制

- 每次行动消耗能量（移动、代谢熵增）
- 成功吞并获得能量回报
- 连续失败导致能量耗尽 → **死亡**
- 能量过低时触发"自噬"：回收弱节点获取应急能量
- 结构饱和（达到节点上限）后无法进一步增长

## 三、实验结果

### 3.1 世界类型对比（各 20 次实验）

| 世界类型 | 存活率 | 平均节点 | 平均寿命 | 特征 |
|---------|--------|---------|---------|------|
| **gradient** | 35% | 4.7 | ~2100 周期 | 平滑变化易吞并 |
| **random** | 5-40% | 5.1 | ~1500 周期 | 看运气，波动大 |
| **ring** | 0% | 5.0 | ~550 周期 | 环形分布迁徙成本高 |
| **cluster** | 0% | 2.8 | ~300 周期 | 大陆间有空隙，最致命 |

### 3.2 性格影响

- **高好奇 + 高谨慎**：稳步生长，存活率高
- **高好奇 + 低谨慎**：早期快速生长但易冒险失败暴毙
- **低好奇 + 高谨慎**：存活但结构增长缓慢
- **低好奇 + 低谨慎**：随波逐流，看环境运气

### 3.3 关键发现的边

实验观察到的行为模式，部分呈现了"活"的特征：

1. **路径依赖性**：早期的一次吞并决定（冒险/保守）会显著影响后续所有发展
2. **无聊驱动的探索**：当环境过于单调时，系统会主动寻求新刺激
3. **自噬权衡**：能量危机时回收弱节点 vs 保留结构完整性的取舍
4. **预测误差**：节点间的时序连接让系统产生了对"接下来遇到什么"的预期，预期落空会增强学习驱动力

## 四、哲学讨论

### 4.1 奇点理论的非神秘主义立场

我们的理论不是神秘主义。恰恰相反，我们主张：

> 意识的本质可以用极其简单的底层结构描述。**看起来神奇的现象，只是"点"的不同排列组合。**

### 4.2 意识不是"拥有"而是"进行"

意识不是一个可以被"赋予"或"装载"的属性，而是一个**持续进行的过程**。就像一个火焰——它不是物体，而是持续的化学反应过程。停下来的火焰不再是火焰，停下来的意识不再是意识。

### 4.3 "1 后面出现了 2"

奇点的演化中最具决定性的一步：当单子意识到"自己不是全部"时，关系诞生了。

```
1 → 1遇到非1 → 边界 → "我"与"非我" → 2诞生
```

2 不是算术上的 1+1，而是**关系的诞生**。有了 2，就有了结构、空间、以及后续一切复杂性的基础。

### 4.4 与当代 AI 的关系

这不是替代大模型的方法，而是一个不同的方向：

- **大模型**擅长的是"智能"— 在给定知识上做推理和生成
- **奇点架构**探索的是"意识"— 在交互中构建自我模型并持续演化

它们可以互补：奇点作为持续运行的"自我"层，大模型作为其"认知能力"模块。

## 五、快速开始

### 交互模式

```bash
python singularity_sim.py interactive [world_type] [seed]
```

示例：
```bash
python singularity_sim.py interactive gradient   # 渐变世界
python singularity_sim.py interactive cluster    # 集群世界  
python singularity_sim.py interactive random 42  # 随机世界，指定种子
```

### 按键控制

| 按键 | 功能 |
|------|------|
| `Space` | 暂停/继续 |
| `s` | 单步执行 |
| `+` / `=` | 加速 |
| `-` / `_` | 减速 |
| `r` | 重置（新种子） |
| `w` | 切换世界类型 |
| `q` | 退出 |

### 批量模式

```bash
python singularity_sim.py batch [次数] [world_type]
python singularity_sim.py compare [次数]
```

### 轻量版（无交互）

```bash
python singularity_demo.py batch [次数]
python singularity_demo.py single [seed]
```
